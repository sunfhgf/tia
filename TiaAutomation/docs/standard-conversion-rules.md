# 标准程序转项目程序规则整理

来源：`C:\Users\Dell\Desktop\PLC手动配置说明.docx`

## 1. 网络配置部分

### 1.1 复制标准模板工程

- 从标准程序复制为项目程序。
- 项目命名格式示例：`DM25120.P1.V20250708`。
- 工具规则：根据项目代号、工位/线体号、日期生成项目名称和输出目录。

### 1.2 安装 GSD 文件

- 打开标准程序后安装 GSD 文件。
- 选择 GSD 文件所在文件夹，全选并安装。
- 工具规则：扫描 `AdditionalFiles/GSD`，列出 GSDML 设备目录，忽略 `*.baiduyun.uploading.cfg`。

### 1.3 添加硬件设备

- 从硬件目录添加伺服、远程 IO 等设备。
- 按配置文件或图纸选择设备型号。
- 工具规则：用 `devices` 配置描述设备名称、类型、PROFINET 名、IP、I/Q 起始地址。

### 1.4 配置网络接口

- 选择设备通讯接口。
- 按电气图纸修改设备名称。
- 按电气图纸修改 IP 地址。
- 工具校验：设备名、PROFINET 名、IP 不允许重复。

### 1.5 伺服通讯报文

- 进入设备视图。
- 选择通讯报文。
- 删除默认报文，替换为项目要求报文。
- 工具规则：用 `servos[].telegram`、`hardwareId`、`telegramAddress` 记录，首版输出计划，暂不自动改硬件。

### 1.6 远程 IO 模块配置

- 进入设备视图。
- 按图纸配置对应型号 IO 模块。
- 按图纸修改 IO 地址。
- 工具规则：用 `ioPoints` 记录设备、通道、地址、标签、注释；校验地址格式和重复。

## 2. 程序编辑部分

### 2.1 伺服参数修改

- 打开伺服逻辑块。
- 根据实际工位/轴名修改名称。
- 每个伺服填写对应参数。
- 注意硬件标识和通讯报文地址。
- 工具规则：生成伺服参数清单，后续可写入标准伺服 DB/FB 调用。

### 2.2 IO 映射

- 打开 IO 映射表。
- 根据图纸填写输入/输出 IO。
- 添加注释，格式按模板编辑。
- 工具规则：输出 PLC 标签 CSV、IO 映射清单和待人工确认项。

### 2.3 气缸/IO 映射

- 打开 IO 映射。
- 注释工位名称。
- 按图纸编写 IO 映射并注释。
- 伺服 Q 点映射到对应变量。
- 气缸手动信号映射到手动变量。
- 到位 I 点映射到对应变量。
- 工具规则：用 `cylinders` 绑定伸出到位、退回到位、伸出阀、退回阀。

### 2.4 特殊气缸/夹爪/吸盘

- 可能没有传感器，需要用动作延时替代到位判断。
- 参数分自动模式、单步模式、初始化模式、屏蔽模式。
- 屏蔽模式延时应短于报警时间。
- 工具规则：气缸配置增加 `mode`、`extendDelayMs`、`retractDelayMs`、`shieldDelayMs`、`alarmTimeMs`。

### 2.5 工位逻辑

- 打开工位逻辑块。
- 按规划修改工位名称。
- 填写工位条件，注意安全风险。
- 工具规则：生成工位清单和安全条件待确认项，不自动替代人工安全判断。

### 2.6 报警逻辑

- 打开报警逻辑块。
- 添加工位注释。
- 把伺服报警、气缸报警添加到对应工位。
- 工具规则：用 `alarms` 配置报警来源、报警文本、等级、工位。

### 2.7 系统 DB / 总变量 DB

- 根据系统需要新增 DB 块。
- 新建系统名称块。
- 把各工位变量添加到系统总变量。
- 模板预留内容按项目实际增删。
- 工具规则：在 `standard-program.json` 记录 DB/块命名规则，后续写入标准块。

### 2.8 电机逻辑

- 打开电机逻辑。
- 按图纸规划修改电机名称。
- 按示例填写电机变量。
- 工具规则：用 `motors` 配置电机类型、运行输出、故障输入、关联设备。

### 2.9 复制设备/工位模板

- 添加新设备或新工位时，从模板复制，粘贴后修改为对应工位。
- 工具规则：后续实现块复制、变量替换、DB 实例生成；首版先输出计划和人工确认清单。

## 3. 首版工具边界

首版工具负责：

- 扫描 GSDML。
- 校验项目配置。
- 生成设备/IO/气缸/伺服/电机/工位/报警计划。
- 生成标签 CSV、气缸映射、手动配置清单。
- 复制模板工程到输出目录。

首版工具暂不负责：

- 自动安装 GSD。
- 自动修改 TIA 硬件组态。
- 自动写入 PLC 块/DB/HMI。
- 自动判断安全互锁条件。
